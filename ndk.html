一、使用ndk-build构建工具进行NDK开发

本文主要讲解了在Windows环境下如何使用ndk-build构建工具来进行NDK开发，以及ndk-build构建工具在Android Stuido中的快捷工具配置。

1.了解一些概念

  • JNI（Java Native Interface）：

    Java原生接口，是Java和其他原生代码语言（例如 C 和 C++）通信的桥梁。

  • NDK（Native Development Kit）：

    原生开发工具集，是一套允许您使用原生代码语言（例如 C 和 C++）实现程序功能的工具集。

  • ABI（Application Binary Interface）：

    应用程序二进制接口，不同的CPU支持不同的指令集，CPU与指令集的每种组合都有其自己的应用二进制接口（或ABI），ABI可以非常精确地定义应用的机器代码在运行时如何与系统交互。

    ABI官方文档

    支持的ABI：armeabi、armeabi-v7a、arm64-v8a、x86、x86_64、mips、mips64

  • CMake：

    Android推荐使用的NDK构建工具，从AS 2.2版本之后开始支持（包含2.2版本）。

生成头文件

  • Name：javah-jni

    工具名称

  • Program：\$JDKPath\$/bin/javah

    javah所在的路径，\$JDKPath\$代表在环境变量中配置的JDK路径。

  • Parameters：-jni -encoding UTF-8 -d \$ModuleFileDir\$\src\main\jni \$FileClass\$

    命令参数：

  • -jni代表生成JNI样式的标头文件，文件名为当前包名+类名（\$FileClass\$）

  • -encoding代表编码格式为UTF-8

  • -d代表指定头文件的输出路径为jni目录（\$ModuleFileDir\$\src\main\jni ）

  • Working directory：\$ModuleFileDir\$\src\main\java

    工作目录，\$ModuleFileDir\$为当前module的路径。

二、使用CMake构建工具进行NDK开发

本文主要讲解了JNI、NDK的一些概念以及在AS 3.0环境下如何使用CMake构建工具来进行NDK开发。

1.环境搭建

安装NDK开发所需的工具

在SDK Tools中安装以下组件：

  • Cmake：NDK构建工具

  • LLDB：NDK调试工具

  • NDK：NDK开发工具集

创建NDK项目

在创建项目时，勾选【Include C++ support】选项，然后一路下一步，到达【Customize C++ Support】设置页：

可以看到三个选项：

  • C++ Standard：C++标准，选择【Toolchain Default】会使用默认的CMake配置。

  • Exceptions Support：支持C++异常处理，标志为 -fexceptions。

  • Runtime Type Information Support：支持运行时类型识别，标志为
    -frtti，程序能够使用基类的指针或引用来检查这些指针或引用所指的对象的实际派生类型。

在这里我们使用默认C++标准，不勾选下面的两个选项，点击【Finish】按钮进入下一个环节。

CMakeLists.txt是CMake的构建脚本，作用相当于ndk-build中的Android.mk，看下CMakeLists.txt：

 1# 设置Cmake最小版本
 2cmake_minimum_required(VERSION 3.4.1)
 3# 编译library
 4add_library( # 设置library名称
 5             native-lib
 6             # 设置library模式
 7             # SHARED模式会编译so文件，STATIC模式不会编译
 8             SHARED
 9             # 设置原生代码路径
10             src/main/cpp/native-lib.cpp )
11# 定位library
12find_library( # library名称
13              log-lib
14              # 将library路径存储为一个变量，可以在其他地方用这个变量引用NDK库
15              # 在这里设置变量名称
16              log )
17# 关联library
18target_link_libraries( # 关联的library
19                       native-lib
20                       # 关联native-lib和log-lib
21                       ${log-lib} )

这是一个基本的CMake构建脚本，更多脚本配置请参考CMAKE手册，看不懂！没关系，这里有中文版的CMAKE手册-中文版。

2.编译FFmpeg

在ffmpeg-3.4.2目录下创建build_android.sh编译脚本，文件名可以随便取，这个脚本是本文中最关键的内容，一起来看下：

 1#!/bin/bash
 2
 3# 设置临时文件夹，需要提前手动创建
 4export TMPDIR="/home/yangle/ffmpeg/ffmpeg-3.4.2/ffmpegtemp"
 5# 设置NDK路径
 6NDK=~/ffmpeg/android-ndk-r14b
 7# 设置编译针对的平台，可以根据实际需求进行设置
 8# 当前设置为最低支持android-14版本，arm架构
 9SYSROOT=$NDK/platforms/android-14/arch-arm
10# 设置编译工具链，4.9为版本号
11TOOLCHAIN=$NDK/toolchains/arm-linux-androideabi-4.9/prebuilt/linux-x86_64
12function build_one
13{
14./configure \
15    --enable-cross-compile \
16    --enable-shared \
17    --disable-static \
18    --disable-doc \
19    --disable-ffmpeg \
20    --disable-ffplay \
21    --disable-ffprobe \
22    --disable-ffserver \
23    --disable-avdevice \
24    --disable-doc \
25    --disable-symver \
26    --prefix=$PREFIX \
27    --cross-prefix=$TOOLCHAIN/bin/arm-linux-androideabi- \
28    --target-os=android \
29    --arch=arm \
30    --sysroot=$SYSROOT \
31    --extra-cflags="-Os -fpic $ADDI_CFLAGS" \
32$ADDITIONAL_CONFIGURE_FLAG
33make clean
34make
35make install
36}
37# 设置编译后文件的输出目录
38CPU=arm
39PREFIX=$(pwd)/android/$CPU
40ADDI_CFLAGS="-marm"
41build_one

2--disable-static

1--prefix=$PREFIX
2CPU=arm

1--target-os=android

10# 设置编译工具链，4.9为版本号
11TOOLCHAIN=$NDK/toolchains/arm-linux-androideabi-4.9/prebuilt/linux-x86_64
12function build_one
13{
14./configure \
15    --enable-cross-compile \
16    --enable-static \
17    --disable-shared \
18    --disable-doc \
19    --disable-ffmpeg \
20    --disable-ffplay \
21    --disable-ffprobe \
22    --disable-ffserver \
23    --disable-avdevice \
24    --disable-doc \
25    --disable-symver \
26    --prefix=$PREFIX \
27    --cross-prefix=$TOOLCHAIN/bin/arm-linux-androideabi- \
28    --target-os=android \
29    --arch=arm \
30    --sysroot=$SYSROOT \
31    --extra-cflags="-Os -fpic $ADDI_CFLAGS" \

四、将FFmpeg移植到Android平台

1.写在前面

我们学习了如何将FFmpeg源码编译成so文件，但是这些so文件还不能直接引用到Android工程中，还需要再次编译加工才能使用，今天就让我们来学习下如何将FFmpeg移植到Android平台，以及在Android项目中如何通过命令行的方式使用FFmpeg。

修改ffmpeg_cmd.c

 1#include <jni.h>
 2#include "ffmpeg.h"
 3JNIEXPORT jint
 4JNICALL
 5Java_com_yl_ffmpeg4android_MainActivity_run(
 6        JNIEnv *env, jclass obj, jobjectArray commands) {
 7    int argc = (*env)->GetArrayLength(env, commands);
 8    char *argv[argc];
 9    int i;
10    for (i = 0; i < argc; i++) {
11        jstring js = (jstring) (*env)->GetObjectArrayElement(env, commands, i);
12        argv[i] = (char *) (*env)->GetStringUTFChars(env, js, 0);
13    }
14    return run(argc, argv);
15}

4.测试

写一个简单的Demo来测试下编译成功的FFmpeg，将一个视频的前100帧截取成一个gif动图，上代码：

 1public class MainActivity extends AppCompatActivity implements View.OnClickListener {
 2    static {
 3        System.loadLibrary("ffmpeg");
 4    }
 5    private ImageView ivGif;
 6    private Button btnConvert;
 7    private ProgressDialog progressDialog;
 8    // 设备根目录路径
 9    private String path = Environment.getExternalStorageDirectory().getAbsolutePath();
10    @Override
11    protected void onCreate(Bundle savedInstanceState) {
12        super.onCreate(savedInstanceState);
13        setContentView(R.layout.activity_main);
14        ivGif = findViewById(R.id.iv_gif);
15        btnConvert = findViewById(R.id.btn_convert);
16        btnConvert.setOnClickListener(this);
17    }
18    @Override
19    public void onClick(View v) {
20        // 截取视频的前100帧
21        final String cmd = "ffmpeg -i " + path + "/video.mp4 -vframes 100 -y -f gif -s 480×320 " + path + "/video_100.gif";
22        // 显示loading
23        progressDialog = new ProgressDialog(this);
24        progressDialog.setTitle("截取中...");
25        progressDialog.show();
26        new Thread() {
27            @Override
28            public void run() {
29                super.run();
30                // 执行指令
31                cmdRun(cmd);
32                // 隐藏loading
33                runOnUiThread(new Runnable() {
34                    @Override
35                    public void run() {
36                        progressDialog.dismiss();
37                        progressDialog = null;
38                        // 显示gif
39                        Glide.with(MainActivity.this)
40                                .load(new File(path + "/video_500.gif"))
41                                .into(ivGif);
42                    }
43                });
44            }
45        }.start();
46    }
47    /**
48     * 以空格分割指令，生成String类型的数组
49     *
50     * @param cmd 指令
51     * @return 执行code
52     */
53    private int cmdRun(String cmd) {
54        String regulation = "[ \\t]+";
55        final String[] split = cmd.split(regulation);
56        return run(split);
57    }
58    /**
59     * ffmpeg_cmd中定义的run方法
60     *
61     * @param cmd 指令
62     * @return 执行code
63     */
64    public native int run(String[] cmd);
65}
